# NeetCode Roadmap Solutions

This repository contains my solutions to problems from the [NeetCode](https://neetcode.io/) roadmap.

I am using the NeetCode roadmap as a structured way to study data structures and algorithms, practise common problem-solving patterns, and prepare for technical interviews.

## Purpose

The main goals of this repository are to:

* Follow a structured data structures and algorithms roadmap
* Learn the most common coding interview patterns
* Solve problems across a wide range of topics
* Improve my ability to identify the correct approach for a problem
* Maintain a record of my progress and solutions
* Revisit and review important problems over time

## Roadmap Topics

Solutions may be organised according to the major NeetCode categories, including:

* Arrays and Hashing
* Two Pointers
* Sliding Window
* Stack
* Binary Search
* Linked List
* Trees
* Tries
* Heap and Priority Queue
* Backtracking
* Graphs
* Advanced Graphs
* One-Dimensional Dynamic Programming
* Two-Dimensional Dynamic Programming
* Greedy
* Intervals
* Mathematics and Geometry
* Bit Manipulation

## Repository Structure

The repository is organised by topic to reflect the NeetCode roadmap.
Each problem folder or solution file may include:

* The problem name
* The solution implementation
* Time complexity
* Space complexity
* Notes about the approach or pattern used

## Languages

Most solutions in this repository are written in:

* Python & C++

Additional languages may be added in the future.

## Learning Approach

For each problem, I aim to:

1. Understand the problem and its constraints.
2. Attempt an independent solution.
3. Identify the relevant data structure or algorithmic pattern.
4. Study alternative or more efficient approaches when necessary.
5. Implement the final solution clearly.
6. Review the problem later to reinforce the concept.

## Progress

This repository will continue to grow as I progress through the NeetCode roadmap.

The objective is not simply to memorise solutions, but to understand why each approach works and how the same pattern can be applied to similar problems.
